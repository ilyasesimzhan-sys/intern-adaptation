# Адаптационная программа стажёров

Сайт для сбора анкет стажёров, формирования учебных групп, отслеживания прогресса, итогового экзамена и рассылки приглашений в WhatsApp. Клиентское React-приложение без собственного сервера. Данные хранятся либо в общем документе Firestore (реальный пилот, доступ с разных устройств), либо в `localStorage` браузера (демо-режим на одном устройстве, если Firebase не настроен).

## Запуск (демо-режим, без настройки)

```powershell
npm install
npm run dev
```

Откройте `http://localhost:5173`. Без настроенного Firebase данные сохраняются только в localStorage текущего браузера — это ок для локальной проверки, но не подходит для реального использования (см. ниже).

- Публичная главная — статус сбора, путь стажёра, список групп после их формирования.
- `#/submit` — анкета стажёра (доступна, пока сбор открыт и группа не сформирована).
- `#/login` — вход для тренеров.

По умолчанию создан один активный аккаунт тренера:

- логин: `trainer1`
- пароль: `password1`

Остальные 9 аккаунтов создаются и редактируются во вкладке «Тренеры» в кабинете.

## Настройка общего хранилища (Firestore) — нужна для реального пилота

Без этого шага анкеты, заполненные с одного компьютера, не увидит тренер на другом компьютере — у каждого браузера будут свои данные. Чтобы данные были общими для всех:

1. Зайдите на [console.firebase.google.com](https://console.firebase.google.com) и создайте новый проект (бесплатно).
2. В разделе **Build → Firestore Database** нажмите «Создать базу данных», выберите любой регион, режим — **тестовый (test mode)**.
3. В **Rules** замените правила на:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /app/state {
         allow read, write: if true;
       }
     }
   }
   ```
   (Это открытый доступ без пароля Firebase — согласуется с моделью безопасности PRD: единственная защита — пароль тренера на входе в кабинет, не более того.)
4. В настройках проекта (⚙️ → **Project settings** → вкладка **General** → раздел **Your apps**) добавьте веб-приложение (иконка `</>`) и скопируйте значения конфигурации.
5. Скопируйте `.env.example` в `.env` и вставьте туда эти значения:
   ```powershell
   copy .env.example .env
   ```
6. Перезапустите `npm run dev` (или пересоберите `npm run build`).

После этого все, кто открывает сайт — с любого устройства, — работают с одними и теми же данными в реальном времени.

## Настройка файлового хранилища (Firebase Storage) — нужна для «Базы знаний»

Раздел «База знаний» (папки и файлы — презентации, документы, картинки) хранит сами файлы не в Firestore, а в Firebase Storage. Без этого шага загрузка файлов не будет работать (зависнет на «Загрузка…», а через 15 секунд покажет ошибку).

1. В том же проекте Firebase зайдите в **Build → Storage** и нажмите «Get started» — включите Storage (бакет уже указан в `.env` как `VITE_FIREBASE_STORAGE_BUCKET`, отдельно создавать его не нужно).
2. В **Rules** замените правила на:
   ```
   rules_version = '2';
   service firebase.storage {
     match /b/{bucket}/o {
       match /knowledge/{allPaths=**} {
         allow read, write: if true;
       }
     }
   }
   ```
   (Тот же открытый доступ без пароля Firebase, что и у Firestore выше — единственная защита остаётся на уровне входа в кабинет тренера.)

После этого главный логин сможет создавать папки и загружать в них файлы во вкладке «База знаний», а остальные тренеры — скачивать их.

## Сборка

```powershell
npm run build
npm run preview
```

`npm run build` создаёт статические файлы в `dist/` — их можно разместить на любом статическом хостинге (Netlify, Vercel, GitHub Pages, любой веб-сервер). Не забудьте передать хостингу те же переменные `VITE_FIREBASE_*` из `.env` (в настройках проекта хостинга, как build-time env vars).

## Важные ограничения (см. PRD, раздел 7)

- Данные общие (shared) для всех, у кого есть ссылка на сайт — это не защищённая корпоративная среда.
- Пароли тренеров — простая защита от случайного входа, не полноценная система безопасности.
- Рассылка WhatsApp — только вручную, по одной кнопке на контакт (ссылки wa.me).
- Совпадающие правки нескольких пользователей одновременно могут перезаписывать друг друга (last-write-wins).
- Без настройки Firebase (см. выше) сайт работает только в пределах одного браузера — анкеты не будут видны с других устройств.

## Структура проекта

- `src/store/StoreContext.jsx` — хранилище данных (Firestore при наличии конфигурации, иначе localStorage) + авторизация тренера.
- `src/lib/firebase.js` — инициализация Firebase (Firestore + Storage) по переменным окружения.
- `src/lib/groupFormation.js` — алгоритм равномерного деления на группы с учётом подразделения и города.
- `src/lib/whatsapp.js` — подстановка шаблона и построение ссылок wa.me.
- `src/pages/HomePage.jsx`, `SubmitPage.jsx`, `LoginPage.jsx` — публичные страницы.
- `src/pages/dashboard/*` — вкладки кабинета тренера.
